Unidad 1

Paradigma del Desarrollo de Aplicaciones Orientadas a Servicios y SOA

Paradigma del desarrollo de aplicaciones orientadas a servicios

El paradigma del desarrollo de aplicaciones orientadas a servicios representa un enfoque moderno para la construcción de sistemas de software, donde las funcionalidades se dividen en unidades independientes llamadas servicios. Cada servicio cumple una función específica y puede ser consumido por diferentes aplicaciones, sin importar el lenguaje o la plataforma utilizada.

Este paradigma surge como respuesta a las limitaciones de los sistemas monolíticos tradicionales, los cuales eran difíciles de escalar, mantener e integrar con otros sistemas. Al separar las funciones en servicios autónomos, se logra una mayor flexibilidad, reutilización y eficiencia en el desarrollo de software.

Los servicios se comunican principalmente mediante protocolos estándar como HTTP y utilizan formatos comunes como XML o JSON para intercambiar información. Esto permite que sistemas heterogéneos trabajen juntos, facilitando la interoperabilidad entre plataformas.

Servicios ofrecidos en la nube

La computación en la nube proporciona recursos informáticos a través de Internet, permitiendo a las organizaciones acceder a infraestructura, plataformas y aplicaciones sin necesidad de adquirir hardware propio.

Infraestructura como Servicio (IaaS): proporciona recursos básicos como servidores virtuales, almacenamiento y redes. Ejemplos incluyen Amazon EC2, Google Compute Engine y Microsoft Azure Virtual Machines. El usuario tiene control sobre el sistema operativo y las aplicaciones.

Plataforma como Servicio (PaaS): ofrece entornos completos de desarrollo donde el proveedor gestiona la infraestructura. El desarrollador solo se enfoca en crear aplicaciones. Ejemplos: Google App Engine, Heroku y Azure App Services.

Software como Servicio (SaaS): entrega aplicaciones listas para usar a través del navegador. Ejemplos comunes son Gmail, Google Drive, Microsoft 365 y Salesforce.

Estos servicios permiten reducción de costos, alta disponibilidad, escalabilidad automática y acceso global.

Características de las aplicaciones orientadas a servicios

Las aplicaciones orientadas a servicios se caracterizan por su bajo acoplamiento, lo que significa que los servicios pueden modificarse sin afectar al sistema completo.

También poseen alta reutilización, ya que un mismo servicio puede ser usado por múltiples aplicaciones.

Son interoperables, permitiendo comunicación entre sistemas desarrollados en distintos lenguajes.

Presentan escalabilidad, pudiendo aumentar recursos según demanda.

Además, cada servicio es autónomo, manteniendo su propia lógica interna.

Aplicaciones Web híbridas (Mashup)

Una aplicación web híbrida o mashup combina información o funcionalidades provenientes de diversas fuentes para crear una nueva aplicación.

Estas aplicaciones integran APIs públicas o privadas para enriquecer la experiencia del usuario. Un ejemplo clásico es combinar mapas de Google con datos empresariales.

Sus características principales incluyen interactividad, consumo de servicios web, actualización dinámica de datos y reutilización de componentes existentes.

Arquitectura Orientada a Servicios (SOA)

SOA es un estilo arquitectónico que organiza el software como un conjunto de servicios independientes que se comunican entre sí mediante mensajes.

Cada servicio representa una función del negocio y puede ser reutilizado por diferentes procesos.

SOA facilita la integración entre sistemas, mejora la escalabilidad y permite una rápida adaptación a cambios empresariales.

Principios de diseño SOA

Los servicios deben ser reutilizables, autónomos, con contratos bien definidos.

Se debe aplicar bajo acoplamiento y alta cohesión.

Los servicios deben ser descubribles y componibles.

Se recomienda ocultar la lógica interna mediante abstracción.

Estándares relacionados

XML: lenguaje de marcado utilizado para estructurar información.

SOAP: protocolo basado en XML para intercambio de mensajes.

WSDL: documento que describe las operaciones de un servicio.

UDDI: registro para publicar y descubrir servicios web.

REST: arquitectura ligera que utiliza métodos HTTP como GET, POST, PUT y DELETE.
